Indecisive

I'm living with two friends, and they’re the nicest people ever, but when it comes to making decisions, they’re a little too nice. A few days ago, they were choosing where to get dinner — to clarify, without me — and they asked me what they should get! Then, we spent over an hour choosing a movie to watch and ended up assigning and picking random numbers.

I found myself wishing that there was an easy way to randomize decisions and decided to build a basic app for myself.

The app opens with a list of Decisions. Navigating into each allows the user to enter choices separated by new lines. Click the Decide button and a random choice is returned to you!
The user can add new decisions to the list, delete from the list, and edit existing names and choices.
Data Persistance
The toughest part of this project was implementing data persistence with NSCoding. I was able to let data persist across multiple runs of the app by creating a persistent path on the file system and adding methods to save and load the data. When the app launches, it loads existing data and when data is modified, the app saves it.

This means the decisions and choice lists inputted will be saved for the next time you use the app. It will remember the restaurants my friends were choosing from this week to be randomized again for next week.
